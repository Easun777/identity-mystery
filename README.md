## 🃏 卡牌图鉴（11 种 × 32 张）

| 卡牌 | 图标 | 数量 | 性质 | 效果 |
|------|:--:|:----:|------|------|
| **第一发现者** | 🔍 | 1 | 主动 | 必须第一个打出，启动游戏 |
| **犯人** | 🦹 | 1 | 主动 | **仅剩此一张时**才能打出 → 犯人阵营获胜 |
| **共犯** | 🕵️ | 0-2* | 主动 | 亮明犯人阵营身份，协助犯人 |
| **侦探** | 🔎 | 1 | 主动 | 指控一人：有犯人且无「不在场证明」→ 非犯人阵营获胜 |
| **不在场证明** | 🛡️ | 3 | 被动 | 在手牌中自动免疫侦探指控 |
| **目击者** | 👁️ | 4 | 主动 | 查看一名其他玩家的**全部手牌** |
| **谣言** | 💬 | 5 | 主动 | 全体从上家随机抽 1 张，刚抽到的不被下家抽 |
| **交易** | 🤝 | 5 | 主动 | 与一名玩家秘密交换 1 张手牌 |
| **情报交换** | 📨 | 5 | 主动 | 每人选 1 张传给下家，新收到的不再传 |
| **普通人** | 🧑 | 4 | 主动 | 无效果，纯粹消耗手牌 |
| **神犬** | 🐕 | 1 | 主动 | 弃掉一人 1 张牌，若弃掉犯人 → 非犯人阵营获胜；否则该玩家收走神犬 |

> \* 共犯数量：3 人局 0 张 / 4-5 人局 1 张 / 6-8 人局 2 张

## 📖 游戏规则

### 准备阶段

- 3-8 名玩家，每人发 **4 张手牌**（隐藏）
- 牌堆共 **玩家数 × 4** 张，从 32 张总牌库中构建
- 必须包含：第一发现者 ×1、犯人 ×1、侦探 ×1、至少 1 张不在场证明
- 共犯数量按玩家人数决定；其余牌随机填充

### 回合流程

```
拿到「第一发现者」的玩家先出牌 → 顺时针轮流 → 每人每回合打出 1 张手牌
                                              ↓
                              打出「犯人」→ 犯人阵营获胜（游戏结束）
```

- 手牌为空的玩家被跳过，由下一位继续
- 打出普通牌 → 效果生效 → 轮到下一位
- 打出犯人 → **即刻结束**

### 胜利条件

| 条件 | 获胜方 |
|------|--------|
| 犯人被成功打出 | 🦹 **犯人阵营**（犯人 + 共犯） |
| 侦探指控犯人（无不在场证明保护） | 🎉 **非犯人阵营** |
| 神犬弃掉犯人牌 | 🎉 **非犯人阵营** |

## 📊 积分规则

每局结束后按以下规则累计积分，多局累加排名：

### 🦹 犯人阵营获胜

| 角色 | 得分 |
|------|:----:|
| 打出犯人的玩家 | **+5** |
| 共犯 | **+3** |
| 其他玩家 | 0 |

> 如果同一玩家既有犯人又是共犯，取最高分 **5 分**。

### 🎉 非犯人阵营获胜

| 情况 | 得分 |
|------|:----:|
| 侦探指控成功（打出侦探者） | **+3** |
| 神犬弃掉犯人（打出神犬者） | **+2** |
| 其他非犯人阵营玩家 | **+1** |
| 犯人阵营玩家 | 0 |

> 共犯即使成功指认犯人或打出神犬，仍按犯人阵营处理 → **0 分**。

## 🛠️ 技术栈

| 层 | 技术 |
|----|------|
| **前端** | HTML5 + CSS3 + 原生 JavaScript |
| **实时通信** | Socket.io（polling 传输，兼容云平台代理） |
| **后端** | Node.js + Express |
| **部署** | Render（免费层，自动 CI/CD） |
| **版本管理** | Git + GitHub |

### 项目结构

```
card_game/
├── server.js           # 游戏服务器（Express + Socket.io）
├── package.json        # 依赖配置
├── index.html          # 单机版（独立运行）
├── start.sh            # 一键启动脚本
├── public/
│   ├── index.html      # 首页导航（联机 / 单机入口）
│   ├── online.html     # 联机版客户端 🎯
│   └── solo.html       # 单机版客户端
└── .gitignore
```

### 架构亮点

- **服务端权威**：所有游戏状态在服务端，客户端仅接收可见信息（自己的手牌、对手的牌数）
- **房间制**：6 位房间号，创建/加入，AI 自动补位
- **投票重开**：所有人同意后才开新局，保持同房间同配置
- **自定义 Socket.io 路径**：`/api/io` 绕过云平台代理拦截

## 🚀 本地运行

```bash
# 安装依赖
cd card_game
npm install

# 启动服务器
node server.js

# 浏览器打开
# 联机版: http://localhost:3000
# 单机版: http://localhost:3000/solo.html
```

## 🤝 联机玩法

1. **房主**创建房间 → 获得 6 位房间号
2. 分享房间号给朋友
3. 朋友们输入房间号加入
4. 房主可添加/移除 AI 凑人数
5. 全员准备 → 房主点「开始游戏」
6. 每局结束后投票「再来一局」
